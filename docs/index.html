💡 What is CoreFx?💡 ¿Qué es CoreFx?

Every JavaFX application re-implements the same plumbing: a singleton to switch screens, a place to stash the logged-in user, helpers to wire up tables, a way to theme scenes, and a validation toolkit. CoreFx is that plumbing, done once and done well. Cada aplicación JavaFX reimplementa la misma fontanería: un singleton para cambiar de pantalla, un lugar donde guardar el usuario conectado, ayudantes para armar tablas, una forma de aplicar temas a las escenas y un kit de validación. CoreFx es esa fontanería, hecha una vez y bien hecha.

It is a small, dependency-free library (JavaFX aside) where every class is final, null-safe by contract, and documented. The runtime-free classes ship with a 47-test JUnit suite. Es una librería pequeña y sin dependencias (más allá de JavaFX) donde cada clase es final, null-safe por contrato y está documentada. Las clases sin dependencia del runtime incluyen una suite JUnit de 47 pruebas.

📦 Installation📦 Instalación

CoreFx is published to Maven Central, so it works out of the box — no extra repositories to configure. Latest version: 1.2.0. CoreFx está publicada en Maven Central, así que funciona de inmediato — sin repositorios extra que configurar. Última versión: 1.2.0. 

<dependency>
  <groupId>io.github.dinamo541</groupId>
  <artifactId>corefx</artifactId>

⚠️
You must add JavaFX yourselfDebes añadir JavaFX por tu cuenta

CoreFx declares JavaFX with provided scope — it does not bundle a JavaFX runtime, so your app controls the version and OS classifier. Add JavaFX 25 (javafx-controls, javafx-fxml) or you will hit NoClassDefFoundError at runtime. CoreFx declara JavaFX con scope provided — no empaqueta un runtime de JavaFX, así que tu app controla la versión y el classifier del SO. Añade JavaFX 25 (javafx-controls, javafx-fxml) o te encontrarás con NoClassDefFoundError en tiempo de ejecución.

🚀 Quick Start🚀 Inicio rápido

A complete, minimal JavaFX application backed by CoreFx:Una aplicación JavaFX mínima y completa apoyada en CoreFx:

import io.github.dinamo541.corefx.navigation.FlowController;
import javafx.application.Application;
import javafx.stage.Stage;

public class MyApp extends Application {

    @Override
    public void start(Stage stage) {
        FlowController flow = FlowController.getInstance();

        flow.initialize(
            stage,                       // the primary stage
            "My Application",            // window title
            "/com/example/views/",       // folder holding your .fxml files
            "/com/example/resources/",   // folder holding other resources
            "/com/example/icon.png",     // application icon
            MyApp.class                  // class used to resolve resources
        );

        // Loads /com/example/views/Home.fxml, builds the scene, and shows it.
        flow.goViewMain("Home");
    }

    public static void main(String[] args) {
        launch(args);
    }
}
