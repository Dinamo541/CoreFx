PersistencePersistencia

A single helper that centralizes access to a shared persistence context (typically a JPA EntityManager) without coupling CoreFx to any persistence provider. Your application supplies a factory; CoreFx stores, lazily creates, caches and hands back the manager as an opaque object. Un único ayudante que centraliza el acceso a un contexto de persistencia compartido (típicamente un EntityManager de JPA) sin acoplar CoreFx a ningún proveedor de persistencia. Tu aplicación provee una fábrica; CoreFx la almacena, crea de forma perezosa, cachea y devuelve el gestor como un objeto opaco.

Why a Supplier?¿Por qué un Supplier?

Referencing jakarta.persistence.EntityManager directly would drag that API onto every consumer's classpath. Instead — like FlowController's theme applier — the consuming app (which already has a JPA provider) passes a Supplier<?>, so the library depends only on the Java platform.Referenciar jakarta.persistence.EntityManager directamente arrastraría esa API al classpath de cada consumidor. En su lugar — igual que el aplicador de temas de FlowController — la app consumidora (que ya tiene un proveedor JPA) pasa un Supplier<?>, de modo que la librería depende únicamente de la plataforma Java.
